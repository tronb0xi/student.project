1. Авторизація реалізується через номер телефону та пароль. Після логіну система видає пару токенів: access та refresh.

POST /api/auth/login/ — Вхід у систему (номер телефону + пароль). Повертає access/refresh токени та роль користувача.

POST /api/auth/refresh/ — Оновлення access токена за допомогою refresh токена.

GET /api/auth/me/ — Отримання даних про поточного користувача (ім'я, роль, доступні філії).

2. Тільки для ролі ADMIN.

GET /api/branches/ - Список усіх філій (активних та архівних).

POST /api/branches/ — Створення нової філії (назва, адреса, місто).

PATCH /api/branches/{id}/ — Редагування даних або переведення філії в статус "archived".

GET /api/subjects/ — Список предметів з фільтрацією за філією.

POST /api/subjects/ — Додавання предмету (перевірка унікальності назви в межах філії).

DELETE/PATCH /api/subjects/{id}/ — Архівування предмету

3. Управління студентами та групами

GET /api/students/ — Список студентів (пошук за ім’ям, фільтр за філією/статусом/групою).

POST /api/students/ — Реєстрація студента (включаючи дані батьків/опікунів).

GET /api/students/{id}/ — Детальна картка студента з історією навчання.

GET /api/groups/ — Список груп філії.

POST /api/groups/ — Створення групи та додавання до неї студентів.

POST/DELETE /api/groups/{id}/members/ — Додавання або видалення студента з групи.

4. Плани підписки (Ціноутворення)

GET /api/subscription-plans/ — Перегляд доступних цінових сіток.

POST /api/subscription-plans/ — Створення нового плану (індивідуальний/груповий, предмети, цінова сітка).

POST /api/student-subscriptions/ — Призначення плану конкретному студенту на певний предмет.

5. Розклад уроків та Шаблони

GET /api/lessons/ — Перегляд розкладу (фільтри: дата, вчитель, група, філія).

POST /api/lessons/ — Створення одиничного уроку (індивідуального або групового).

POST /api/lesson-templates/ — Створення шаблону для регулярних уроків (генерація занять на період).

PATCH /api/lessons/{id}/ — Редагування уроку або зміна статусу (SCHEDULED / COMPLETED / CANCELLED).

6. Відвідуваність та Звіти

POST /api/lessons/{id}/attendance/ — Фіксація присутності/відсутності студентів та додавання коментарів.

GET /api/reports/attendance-history/ — Історія відвідуваності окремого студента.

GET /api/reports/branch-statistics/ — Загальна статистика філії (кількість активних студентів, % відвідуваності).

GET /api/reports/teacher-schedule/ — Формування звіту по графіку конкретного вчителя.
